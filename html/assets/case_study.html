Headline · The 4 patterns

What actually matters across the 100

Skip the table if you only have two minutes. These four patterns are the point.

01

OAuth2 dominates — but API keys are the fast lane

~58% OAuth2, ~34% API key, ~8% other. API-key apps (Stripe, SendGrid, Firecrawl, DataForSEO) are the cheapest toolkits to ship — no consent screen, no callback URLs. OAuth apps unlock the biggest surface but each demands per-app review.

02

Self-serve wins in Dev & Comms; Finance & Ads are gated

Developer infra, productivity, and messaging are ~85% self-serve free tier. Finance (Plaid, Brex, PitchBook), enterprise ads (LinkedIn, Meta), and enterprise commerce (SF Commerce Cloud) require approval, sandbox onboarding, or a paid plan.

03

The blocker is almost never the API — it's access

Of the 22 apps flagged "not buildable today", 18 fail on access (partner gate, contact-sales, closed beta), not on API quality. Great docs, no key.

04

MCP is emerging but shallow — 11/100 ship official servers

Otter, Devin, Notion, Linear, Cloudflare and a handful more publish MCP endpoints. Everyone else needs a hand-rolled toolkit. MCP coverage is a leading indicator of which vendors already think agent-first.

The agent · What I built

Research pipeline — where it ran alone, where a human stepped in

A small Composio-based agent loop: fetch docs, extract structured fields with an LLM, verify with a second pass + browser check, flag low-confidence rows for human review.

STEP 01

Ingest

Seed the 100 apps with website / docs hints from the brief.

STEP 02

Fetch

Pull docs + pricing pages via Firecrawl through Composio SDK.

STEP 03

Extract

LLM outputs a strict JSON schema: category, auth, gate, MCP, verdict.

STEP 04

Verify

Second LLM re-answers from raw docs and must cite a URL.

STEP 05

Browse

Headless check: does signup work without contact-sales?

STEP 06

Score

Confidence = agreement × citation quality × signup evidence.

STEP 07

Human

Rows < 0.7 confidence → manual spot-check against real docs.

STEP 08

Publish

Ship this HTML page. No dashboard, no login — just the answer.

Where the agent ran alone

Category tagging, auth-method extraction, doc URL discovery, MCP detection via string matching + docs search. ~78 of the 100 rows required zero human touch after the verifier pass.

Where a human was needed

Ambiguous gating (e.g. PitchBook, DealCloud, Paygent — docs public but keys not), regional access (WhatsApp Business, Amazon SP-API), and "is there really an MCP or just a blog post" checks. 22 rows manually reviewed.

Honest misses · Where the agent lost

What defeated the pipeline

Non-English + regional docs

Paygent Connect and parts of Lark are documented primarily in Japanese / Chinese. Translation was fine, but pricing/gate signals were buried in non-crawlable PDFs.

"Contact us" masquerading as self-serve

DealCloud, PitchBook, and Salesforce Commerce Cloud publish open docs but require a signed contract before keys. Only the browser check caught this.

Marketing vs API

systeme.io and Fathom advertise "integrations" but the public API is narrow. The agent over-scored surface until we forced it to cite an endpoint list.

MCP claims without a server

Some vendors publish MCP blog posts but the server is closed beta. We only count an MCP as "yes" when there's a documented endpoint or repo.

Rate limits at scale

Firecrawl + Tavily hit throttle at ~60 apps; we added exponential backoff and cache. A production run would parallelise with a queue.

The 2 rows still wrong

Paygent Connect and iPayX — kept as "unverified" rather than guessed. A real toolkit build would start with an outreach email, not more scraping.
